# AquaPulse: 評価指標（KGI/KPI）設計方針

## 1. 究極のKGI（生体の死）を見送る戦略的理由
水槽管理の究極の目的は「生体の生存」だが、データ分析の実務観点から以下の理由でこれを直接的なKGIには設定しない。
- **データスパースネス（疎性）**: 発生頻度が低すぎて、モデルの学習や統計的な有意差の検証が困難である。
- **交絡因子（ノイズ）**: 生物特有の「寿命」や「内部疾患」による死と、環境悪化による死の切り分けが不可能に近い。

**💡 ポートフォリオとしてのアピールポイント:**
ビジネスの実務（LTV予測や完全な顧客離反など）においても、究極のKGIは「遅行指標」かつ「スパース」であるため日々の最適化には使えない。この「実データの限界に気づき、アクション可能な先行指標（プロキシ）へ問題を再定義した」という意思決定プロセス自体を、データアナリストとしての課題解決能力の証明とする。

---

## 2. プロキシKGI（代替指標）のアイデア

実務におけるビジネス価値（コスト削減、リスク管理、SLA向上）に翻訳しやすい以下の3つを候補とし、データが蓄積した段階で最も因果関係が明確なものを最終的なKGIとして採用する。

### アイデアA: 「ボラティリティ（水質・水温の急変）」の最小化
- **ロジック**: 生体への最大のダメージ（死のトリガー）は、環境の「乱高下」によるショックである。
- **プロキシKGI**: 「1日の水温の分散（Variance）」 または 「pHの最大変動幅（Max - Min）」
- **ビジネス翻訳**: 金融やマーケティングにおける**「リスク（分散）の最小化」**。
- **評価ストーリー**: 「外気温の変化が水温に与える遅延効果をモデル化し、予測に基づくアラート・制御を行った結果、水温のボラティリティをX%低減し、安全な環境（リスク最小化）を構築した」

### アイデアB: 「運用コスト（水換え工数）」の最小化
- **ロジック**: ターゲットを「魚の健康」から「人間側の運用効率化」へシフトする。
- **プロキシKGI**: 「水質（TDS等）が安全閾値に達するまでの日数（＝水換えのインターバル）」
- **ビジネス翻訳**: オペレーションの**「コスト削減・リソース最適化」**。
- **評価ストーリー**: 「経験則による『なんとなく週1回の水換え』を、データのトレンド予測に基づき『生体の健康（ガードレール指標）を担保した上で10日に1回へ延長』し、年間の作業工数を30%削減した」

### アイデアC: 「異常状態の滞在時間 (Time-in-Abnormal State)」の最小化
- **ロジック**: 監視システム（IoTアラート）を導入したことによる「復旧スピードの向上」を評価する。
- **プロキシKGI**: 「水温が安全域（例: 24℃未満、または28℃以上）を逸脱していた月間累積時間」
- **ビジネス翻訳**: SaaSビジネスやインフラにおける**「ダウンタイムの最小化（SLAの向上）」**。
- **評価ストーリー**: 「異常検知システムとLINEアラートを構築した結果、機材トラブル等による異常状態の放置時間を月間X時間からY時間に短縮し、致命的な障害を未然に防いだ」
