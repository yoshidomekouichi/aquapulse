# ハードウェア試行ログ

改善計画の実施結果・トライアンドエラーを記録する。  
**計画**は [improvement-plan.md](improvement-plan.md) を参照。

---

## 記録の仕方

各試行ごとに以下を記録する：

| 項目 | 内容 |
|------|------|
| **日付** | 実施日 |
| **タスク** | improvement-plan のタスク番号 |
| **やったこと** | 具体的な変更・配線・設定 |
| **結果** | 成功 / 失敗 / 一部成功 |
| **学び** | 分かったこと・注意点 |
| **次のステップ** | 次に試すこと、または wiring 更新 |

---

## 試行履歴

### 2026-03-15: TDS 水槽内 0ppm の原因調査

| 項目 | 内容 |
|------|------|
| **タスク** | 背景調査 |
| **やったこと** | 技術レポート受領。寄生静電容量・分流メカニズムの理解。CH1- を GND に接続（差動入力の負側安定化） |
| **結果** | 小瓶では正常値、水槽では 0ppm のまま。CH1- 接続は小瓶測定の安定化には寄与 |
| **学び** | 単純な GND 接続では水槽内の分流は解決しない。ガルバニック絶縁が必須 |
| **次のステップ** | DFR0504 アイソレータ導入（improvement-plan タスク1） |

### 2026-03-15: 暫定運用（瓶測定）の整備

| 項目 | 内容 |
|------|------|
| **タスク** | アイソレータ導入までの運用方針 |
| **やったこと** | `measure_tds_bottle.py` 作成。1日1回瓶で測定する運用を採用。`.env` 変更・docker compose 不要で実行可能。`--save` で DB に保存 |
| **結果** | スクリプト完成。improvement-plan に暫定運用を追記 |
| **学び** | プローブの通電時間を最小化することで電極劣化を遅らせられる |
| **次のステップ** | 日常運用で瓶測定を実施。DFR0504 到着後に本格導入 |
