### **配線手順（ステップバイステップ）**

#### **ステップ1: ESP32をブレッドボードに挿す**

```
1. ブレッドボードの中央の溝を確認
2. ESP32を溝をまたぐように挿す
3. ピンがしっかり入ったか確認
4. 左右のピンが使えるか確認

注意:
  - USB端子が上または下に来るように
  - ピンを曲げないように注意
  - 力を入れすぎない
```

#### **ステップ2: 電源レールの接続**

```
1. ジャンパーワイヤー（赤）
   ESP32の 3V3 → ブレッドボードの + レール

2. ジャンパーワイヤー（黒）
   ESP32の GND → ブレッドボードの - レール

確認:
  - 赤は + レール（通常は赤いライン）
  - 黒は - レール（通常は青または黒いライン）
```

#### **ステップ3: DS18B20の接続**

```
1. DS18B20の3本の線を確認
   - 赤（VDD）
   - 黒（GND）
   - 黄または緑（DATA）

2. 接続
   赤（VDD）  → + レール
   黒（GND）  → - レール
   黄（DATA） → ESP32 GPIO4

3. プルアップ抵抗（4.7kΩ）
   DATA線と 3V3 の間に接続
   
   ブレッドボード上で:
   ┌──4.7kΩ──┐
   │         │
   3V3     GPIO4
   
確認:
  - 赤と黒を逆にしないこと！
  - プルアップ抵抗は必須
  - モジュール内蔵の場合は不要
```

#### **ステップ4: MCP3424の接続**

```
1. MCP3424のピンを確認
   - VDD（電源）
   - GND（グランド）
   - SDA（データ）
   - SCL（クロック）
   - CH1+（チャンネル1 正）
   - CH1-（チャンネル1 負）

2. 電源接続
   VDD → + レール
   GND → - レール

3. I2C接続
   SDA → ESP32 GPIO21
   SCL → ESP32 GPIO22

4. TDSプローブ接続
   CH1+ → TDSプローブ 赤（信号線）
   CH1- → TDSプローブ 黒（GND）

確認:
  - I2Cアドレス: 0x68
  - プルアップ抵抗内蔵（ESP32側）
```

## 🧪 **動作確認**

### **ステップ1: 配線チェック**

```
チェックリスト:
  □ 電源（3V3, GND）が正しい
  □ DS18B20のDATA線がGPIO4
  □ MCP3424のSDAがGPIO21
  □ MCP3424のSCLがGPIO22
  □ プルアップ抵抗が入っている
  □ ショート（短絡）がない
```

### **ステップ2: 電源投入**

```bash
# USB接続
1. ESP32にUSBケーブルを接続
2. PCに接続
3. LEDが点灯するか確認

# 電圧確認（テスター使用）
4. 3V3ピン: 3.3V
5. 5Vピン: 5V
6. GND: 0V
```

### **ステップ3: I2Cデバイス確認**

```bash
# MicroPythonで確認（後のセクション）

from machine import I2C, Pin
i2c = I2C(0, scl=Pin(22), sda=Pin(21))
devices = i2c.scan()
print(hex(devices[0]))  # 0x68 が表示されればOK
```

### **ステップ4: 1-Wireデバイス確認**

```bash
# MicroPythonで確認

from machine import Pin
import onewire
ow = onewire.OneWire(Pin(4))
devices = ow.scan()
print(devices)  # デバイスIDが表示されればOK
```

## 🔍 **トラブルシューティング**

### **ESP32が認識されない**

```
症状: PCに接続してもデバイスが見えない

原因:
  1. USBケーブルが充電専用
  2. ドライバ未インストール
  3. USB端子の不良

対策:
  1. データ通信対応ケーブルを使用
  2. CP210xドライバをインストール
  3. 別のUSBポートを試す
```

### **センサーが認識されない**

```
症状: i2c.scan()やow.scan()が空

DS18B20の場合:
  1. プルアップ抵抗の確認
  2. DATA線の接続確認
  3. 電源（3.3V）の確認

MCP3424の場合:
  1. I2Cアドレス確認（0x68）
  2. SDA/SCLの接続確認
  3. プルアップ抵抗（内蔵）
```

### **電圧が安定しない**

```
症状: センサー値がおかしい

原因:
  1. 電源容量不足
  2. 配線が長すぎる
  3. ノイズ

対策:
  1. USBアダプタ使用（5V 1A以上）
  2. 配線を短くする
  3. コンデンサ追加（100nF）
```
