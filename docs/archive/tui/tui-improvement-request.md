# TUI ダッシュボード 改善リクエスト

**作成日**: 2026-03-17  
**状態**: 承認待ち

---

## 1. 問題点

| 項目 | 内容 |
|------|------|
| **ドットが荒い** | グラフの点字ドットが粗く、Dolphin のような細かいドットになっていない |
| **見づらい** | かなり見づらい |
| **レイアウト** | 7 インチ画面（800x480）に沿った最適化になっていない可能性 |

---

## 2. 原因の整理

| 原因 | 説明 |
|------|------|
| **点字の物理解像度** | 点字は 1 文字あたり 2×4 ドット。`height=4` 行 = 縦 16 ドット、`width=40` 文字 = 横 80 ドット。この密度が 7 インチでは粗く見える |
| **フォントサイズ** | 7 インチコンソールのデフォルトフォントは読みやすさ優先で大きめ。1 文字あたりのピクセルが多く、点字 1 ドットが大きく表示される |
| **レイアウトの固定値** | `GRAPH_WIDTH=40`、`height=4` 等が固定。ターミナルセル数に依存するため、800x480 の実効解像度（フォント依存）と合っていない可能性 |

---

## 3. 要望

| 要望 | 備考 |
|------|------|
| **Dolphin のような細かいドット** | 荒いドットではなく、細かく見やすいドット表現 |
| **見やすさの向上** | 現状より視認性を高めたい |
| **線風の棒グラフは不要** | ▁▂▃▄▅▆▇█ 等のスパークライン風は採用しない（ユーザー指示） |

---

## 4. 合理的な改善案

要望（細かいドット・見やすさ・線風棒グラフは不可）を満たすため、以下の案を提案する。

### 案 A: 点字の解像度を上げる【推奨】

| 項目 | 内容 |
|------|------|
| **内容** | `height=4` → `height=6` または `height=8` に増やす。`GRAPH_WIDTH=40` → `50` 前後に増やす |
| **根拠** | 同じ点字方式のまま、縦横のドット数を増やせば「細かいドット」に近づく。要望に沿う |
| **懸念** | パネルが大きくなり、4 分割レイアウトで収まらなくなる可能性。その場合はパディングやフォント調整で調整 |

### 案 B: フォントサイズの最適化（案 A と併用可）

| 項目 | 内容 |
|------|------|
| **内容** | 7 インチ用にコンソールフォントをやや小さくする（`console-setup` や `setfont`） |
| **根拠** | 同じ点字数でも 1 文字あたりの表示が小さくなり、相対的に「細かいドット」に見える |
| **懸念** | システム設定の変更。ラベル等の文字が小さくなりすぎないよう、フォント選定が必要 |

### 案 C: 点字以外のドット系文字の検討

| 項目 | 内容 |
|------|------|
| **内容** | 点字（U+2800 系）以外で、より細かく見えるドット表現を探す（例: 中黒 `·`、細い罫線の組み合わせ等） |
| **根拠** | フォントによっては点字より細く描画される文字がある可能性 |
| **懸念** | 線グラフの連続性を保ちつつ「細かいドット」を実現できるか不明。要検証 |

### 案 D: レイアウトの 7 インチ最適化（別軸）

| 項目 | 内容 |
|------|------|
| **内容** | ターミナルサイズを取得し、800x480 相当（例: 約 100×30 セル）に合わせて `GRAPH_WIDTH` 等を動的調整 |
| **根拠** | 画面に沿ったレイアウトになり、余白やはみ出しを減らせる |
| **懸念** | フォント設定によって実効セル数が変わるため、環境依存になりうる |

---

## 5. 提案する実施順序

1. **案 A** を先行実施（解像度アップ）。効果が不十分なら **案 B** を検討
2. 必要に応じて **案 D** でレイアウトを調整
3. **案 C** は A/B で改善しない場合の代替案として検討

---

## 6. 承認フロー

1. 上記の案・実施順序を確認
2. ユーザーが許可
3. 実装
