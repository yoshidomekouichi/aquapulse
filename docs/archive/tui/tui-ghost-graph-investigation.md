# ゴーストグラフ問題 原因調査レポート（2026-03 追跡）

## 現状（修正後の悪化）

- **グラフが表示されない**：正しいフルサイズのグラフが出なくなった
- **ゴーストは残る**：小さな fallback サイズのグラフが依然として表示される
- **最悪の状態**：以前より悪化

---

## 原因分析

### 1. 閾値が厳しすぎる（グラフ非表示の主因）

**直前の修正**：`on_resize` で `event.size.width < 60` または `event.size.height < 15` のとき即 `return` するようにした。

**問題**：`docs/tui-terminal-font-setup.md` より、800x480 での目安は：

| フォント | 全体 | グラフパネルあたり |
|----------|------|---------------------|
| 6x12 | 66行×133桁 | **高さ約15行、幅約65桁** |
| 8x16 | 30行×100桁 | 高さ約7行、幅約50桁 |
| 16x32 | 15行×50桁 | 高さ約3行、幅約25桁 |

- 6x12 でもパネルあたり **幅65・高さ15** 程度で、パディング・マージンを引くと **幅59・高さ14** 前後になる可能性が高い
- 閾値 60×15 だと、境界付近で `return` し、**一度もキャッシュされず描画されない**
- 8x16 / 16x32 ではさらに小さく、ほぼ確実に閾値未満になる

**結論**：閾値 60×15 は実機（特に Pi Touch 800x480）では高すぎる。

### 2. 閾値未満時の挙動

```python
if event.size.width < 60 or event.size.height < 15:
    return  # キャッシュも _redraw() も呼ばない
```

- 閾値未満の Resize が来ると何もしない
- その結果 `_valid_width` / `_valid_height` が 0 のまま
- `update_data` → `_redraw()` でも `_valid_width <= 0` で即 return し、描画されない

### 3. ゴーストが残る理由（仮説）

- 閾値通過時に一度だけキャッシュ・描画されるケースがある
- その後のレイアウト再計算で小さい Resize が来るが、`return` でキャッシュは更新しない
- ただし、**別の経路**（例：親の `update(layout=True)` や他ウィジェットの更新）で、古い小さい描画が残る可能性がある
- あるいは、**初回レイアウトで小さいサイズが先に来る**場合、その時点でキャッシュ・描画され、ゴーストが固定される

### 4. layout=False の影響

- グラフ用 Static の `update(plot_text, layout=False)` は妥当
- ただし、ヘッダー・ステータス・最新値の `update` が `layout=True` のままなら、それらがレイアウト再計算を起こし、Resize の順序や値に影響する可能性がある

---

## 明日の改善方針

### A. 閾値の見直し（最優先）

1. **閾値を下げる**  
   - 例：`_MIN_VALID_WIDTH = 40`, `_MIN_VALID_HEIGHT = 8`  
   - 8x16 でも「幅50・高さ7」程度は確保できる想定

2. **閾値未満でも return しない**  
   - 閾値未満のときは **キャッシュだけ更新しない**（既存のキャッシュを維持）  
   - `_redraw()` は呼ぶ（既存キャッシュで再描画）

3. **デバッグログの一時追加**  
   - `on_resize` で `event.size.width`, `event.size.height` をログ出力  
   - 実機で実際の値を確認してから閾値を決める

### B. 閾値ロジックの変更案

```python
def on_resize(self, event: Resize) -> None:
    w = max(0, event.size.width - 4)
    h = max(0, event.size.height - 2)
    # 閾値以上の場合のみキャッシュを更新（小さい値で上書きしない）
    if w >= self._MIN_VALID_WIDTH and h >= self._MIN_VALID_HEIGHT:
        self._valid_width = w
        self._valid_height = h
    # キャッシュが有効なら常に _redraw()（閾値未満でも return しない）
    self._redraw()
```

- 閾値未満の Resize が来ても `return` しない
- キャッシュは「十分大きいときだけ」更新
- 既存キャッシュがあれば `_redraw()` で再描画

### C. フォント設定の確認

- 実機で `6x12` が有効か確認
- `/etc/default/console-setup` の `FONTSIZE` を確認

---

## まとめ

| 項目 | 内容 |
|------|------|
| グラフ非表示 | 閾値 60×15 が高すぎて、実機でキャッシュが一度も設定されない |
| ゴースト残存 | 閾値通過時の描画や、他ウィジェットの layout 更新が影響している可能性 |
| 推奨対応 | 閾値の引き下げ、閾値未満でも `return` しないロジックへの変更、実機でのサイズログ取得 |
