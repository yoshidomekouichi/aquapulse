# TUI ディスプレイ トラブルシュートログ（2026-03-17）

## 概要

Raspberry Pi 5 + 公式 7" タッチディスプレイ（800x480 DSI）で AquaPulse TUI を表示。  
**症状**: ディスプレイが真っ暗 → **解決**: `dtoverlay=vc4-kms-dsi-7inch,dsi1` にて復旧。

| 項目 | 内容 |

## 3. 判明した技術的なポイント

### 3.1 バックライト点灯・画面真っ黒の意味

- **バックライト ON** → 電源・I2C は通っている。ディスプレイは部分的に検出されている
- **画面真っ黒** → DSI の映像信号がパネルに届いていない
- **切り分け**: 懐中電灯テスト（暗い部屋で光を当ててかすかに画像が見えるか）でバックライト故障と信号不良を区別可能

### 3.2 Pi 5 の DSI ポート

- Pi 5 には 2 つの MIPI ポート（CAM0/CAM1）があり、両方カメラ・ディスプレイ対応
- **dsi0 と dsi1 は物理的な接続先に依存**。接続しているコネクタに応じて `dsi0` を指定するか `dsi1` を指定するかが変わる
- 本環境では **dsi1** が正しいポートだった

### 3.3 ケーブルについて

- Pi 5 のアダプタケーブルは**カメラ用とディスプレイ用で別物**（配線が異なる）
- 本件では以前から表示されていたため、ケーブル種類は問題ではなかった
- 端子・ポート番号も問題ではなく、**DSI 映像パイプラインの接続**が原因だった

## 4. 残課題（明日対応予定）

### 4.1 ブートログがずっと流れる

**症状**: 起動直後のメッセージが止まらない。TUI が表示されるまでブートログが画面を埋める。

**対策案**:
- `/boot/firmware/cmdline.txt` に `quiet loglevel=3` を追加してカーネルメッセージを抑制
- `console=tty3` を追加してコンソール出力を tty3 にリダイレクト（tty1 を TUI 専用にする）
- または `ExecStartPre` の `sleep 15` を延長（例: 30 秒）

**実施例**（cmdline.txt は 1 行。既存の行末にスペース区切りで追加）:
```bash
# バックアップ
sudo cp /boot/firmware/cmdline.txt /boot/firmware/cmdline.txt.bak

# quiet が無い場合のみ追加（重複防止）
grep -q 'quiet' /boot/firmware/cmdline.txt || sudo sed -i 's/$/ quiet loglevel=3/' /boot/firmware/cmdline.txt

sudo reboot
```

### 4.2 レイアウトが 7 インチ画面に沿っていない

**症状**: グラフがガッタガタ。画面全体にフィットしていない印象。

**原因**: TUI は**ターミナルセル（文字数×行数）**でレイアウト。7 インチ 800x480 の実効サイズはフォントやコンソール設定に依存。`dashboard.py` は `GRAPH_WIDTH=40` 等の固定値を使用。

**対策案**:
- フォントサイズの調整（`console-setup` や `setfont`）
- ターミナルサイズに応じた動的レイアウト
- 800x480 ピクセルに最適化した固定レイアウトの再調整

- **ダッシュボードのブラッシュアップ**: レイアウト調整、グラフの見やすさ改善
